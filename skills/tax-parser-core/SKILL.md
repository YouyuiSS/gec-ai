## What I do

- Provide shared parser runtime for pack skills.
- Run parser, validator, diff, follow-up execution, and quality gate.
- Support pack-scoped execution through `--pack-dir`.

## When to use me

- You are creating a new tax parser pack.
- You need to run tests or quality gate for an installed pack.
- You need to monitor official sources for a pack.
- You need to execute follow-ups generated by pack monitoring.

## Runtime contract

- Install this skill together with at least one `tax-pack-*` skill.
- Pack commands should pass `--pack-dir <pack-skill-dir>` to the core scripts.
- The pack owns sources, profiles, overlays, and baselines.
- The core owns shared runtime and gate behavior.

## Core commands

Run a pack parser:

```bash
python scripts/run_tax_parser.py \
  --pack-dir <pack_dir> \
  --pdf <pdf_path> \
  --outdir <outdir>
```

Test a pack profile:

```bash
python scripts/test_extractor.py \
  --pack-dir <pack_dir> \
  --pdf <pdf_path> \
  --extractor <profile_name> \
  --outdir <outdir>
```

Monitor a pack's official sources:

```bash
python scripts/monitor_official_sources.py \
  --pack-dir <pack_dir> \
  --outdir <outdir>
```

Run pack follow-ups:

```bash
python scripts/run_source_followups.py \
  --pack-dir <pack_dir> \
  --followups <followups.json> \
  --runner test_extractor \
  --outdir <outdir>
```

Run the pack quality gate:

```bash
python scripts/quality_gate.py \
  --pack-dir <pack_dir> \
  --pdf <pdf_path> \
  --extractor <profile_name> \
  --outdir <outdir>
```
